================================================================================
geo-ncbi-fetch — GEO 数据集元数据批量提取 Skill
================================================================================

版本: v1.2.0
适用: scRNA-seq / 生信数据集整理 / GEO 数据库注释工作流
依赖: Python 3.x + Biopython + conda 环境管理

================================================================================
一、功能概述
================================================================================

从 NCBI GEO 数据库批量提取数据集元数据，输出结构化 JSON，支持断点续跑。
配合 academic-translate skill 可实现中文学术翻译，最终整合到 Excel 数据集清单。

输入: GSE/GSM/GPL 编号列表（直接输入、Excel 文件、笔记内容、JSON 文件）
输出:
  - JSON 文件 + 翻译文件 + Excel 表格（一键输出）

  JSON 字段:
  ┌─────────────┬──────────────────────────────────────────┐
  │ 字段         │ 说明                                      │
  ├─────────────┼──────────────────────────────────────────┤
  │ accession   │ GEO 编号                                   │
  │ title_en    │ 数据集英文标题（来自 NCBI）                  │
  │ summary_en  │ 实验设计描述（英文）                        │
  │ gdsType     │ 实验类型（芯片/测序/甲基化分析等）           │
  │ n_samples   │ NCBI 记录的样本数量                         │
  │ taxon       │ 物种（Homo sapiens / Mus musculus）        │
  │ PDAT        │ 发布日期（YYYY/MM/DD）                      │
  │ supplFiles  │ 补充文件类型列表（CEL, MTX, TSV, CSV 等）    │
  │ pmids       │ 关联的 PubMed ID 列表                       │
  │ sample_types│ 从摘要中提取的样本类型（中文标签）           │
  │ geo_url     │ GEO 数据库页面链接                          │
  └─────────────┴──────────────────────────────────────────┘

三、核心特性
================================================================================

1. 环境自检（Step 0）
   - 列出系统所有 conda 环境，让用户选择
   - 自动检测 Biopython 是否安装
   - 缺失时通过清华镜像安装: pip install biopython -i https://pypi.tuna.tsinghua.edu.cn/simple

2. 批量提取 + 断点续跑
   - ≤5 个编号: 直接执行，一次完成
   - >5 个编号: 自动拆分批次（默认 5 个/批），每批写 checkpoint 文件
   - 中断后可通过 --start-batch N 从第 N 批恢复，已完成批次不重跑

3. 速率控制
   - 遵守 NCBI 官方速率限制（≤3 req/s）
   - 顺序执行，不触发 HTTP 429 封禁
   - sleep 间隔 0.35s（可根据网络调整）

4. 安全设计
   - 仅提供 GEO 数据库页面 URL，不爬取 FTP 下载链接
   - 翻译由 academic-translate skill 独立完成，职责清晰
   - 错误不中断流程：not_found/error 标记后继续

================================================================================
四、使用方式

4.1 通过 Claude Skill 调用（推荐）
================================================================================

直接对 Claude 说：

  "用 geo-ncbi-fetch skill 提取这批 GEO 数据的元数据:  GSE76039"

  "从 数据集.xlsx 中筛选 GEO 来源的 GSE，提取元数据"

  "把笔记「甲状腺数据集」里所有 GEO 编号的元数据拉下来"

Claude 会按顺序执行：
  1) 询问 conda 环境 → 2) 解析编号 → 3) 提取元数据 → 4) 输出 JSON

4.2 命令行直接调用
================================================================================

# 基础用法（≤5 个编号）
python scripts/fetch_geo.py \
  --input "GSE76039" \
  --output meta.json \
  --email your@email.edu

# 批量模式（>5 个编号，自动拆分批次）
python scripts/fetch_geo.py \
  --input accs.txt \
  --output meta.json \
  --email your@email.edu \
  --batch-size 5

# 断点续跑（从第 3 批开始，跳过 0-2）
python scripts/fetch_geo.py \
  --input accs.txt \
  --output meta.json \
  --batch-size 5 \
  --start-batch 3

五、与其他 Skill 协作
================================================================================

完整工作流（全内置）:

  原始数据列表 / 已有Excel
       │
       ▼
  ┌──────────────────────┐
  │  geo-ncbi-fetch       │
  │  ① 解析编号 / Excel   │
  │  ② 提取 NCBI 元数据   │
  │  ③ academic-translate │
  │  ④ build_excel.py     │
  │     ├─ 新建模式        │
  │     └─ 追加模式        │
  │  输出: .xlsx           │
  └──────────────────────┘

  触发策略（自动选择）:
  - 提供 GSE 编号 → 新建 Excel
  - 提供已有 .xlsx → 追加到原表（输出 _annotated.xlsx）

六、速率与规模参考

  ┌──────────────┬─────────────┬──────────────────┐
  │ 编号数量      │ 模式         │ 预计耗时          │
  ├──────────────┼─────────────┼──────────────────┤
  │ 1-5 个       │ 直接模式     │ ~2 秒             │
  │ 10 个        │ 批处理(5×2)  │ ~4 秒             │
  │ 50 个        │ 批处理(5×10) │ ~20 秒            │
  │ 100 个       │ 批处理(5×20) │ ~40 秒            │
  └──────────────┴─────────────┴──────────────────┘

  注意: 以上为无 API Key 情况（3 req/s）。注册免费 NCBI API Key 可缩短约 3 倍。

================================================================================
七、局限性

  - 仅返回 NCBI Entrez 中索引的元数据，原始表达矩阵需通过 GEO 页面手动下载
  - 不提供 FTP 直接下载链接（仅 geo_url 数据库页面链接）
  - 仅支持 GEO（GSE/GSM/GPL），不支持 GSA/ENA/CNGB/Zenodo 等来源
  - NCBI 偶尔索引延迟，新提交的数据集可能需要 1-2 天才能查到
  - 速率限制为 per-IP，不支持多进程并行加速
